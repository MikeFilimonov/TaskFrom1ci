Bank statement processing settings

The format of an exported bank statement depends on the bank client system.

To be able to import data from a bank statement file, you have to add and configure a data processor that supports this data format. Typically, for each file format you have a separate data processor in 1C:Drive.

To view and edit the list of data processor settings, click Cash management > Bank statement processing settings.

Each list item represents data processor settings and has the following fields:

  • Description. The name of the data processor settings.
  • Data processor. A reference to a data processor available in 1C:Drive. For more information, see ???
  • Encoding. The encoding of source bank statement files.
  • File extensions. The supported extensions of source bank statement files. Use the following template:

    Format_name1|*.extension1|Format_name2|*.extension2

    Example:

    XML file (*.xml)|*.xml|Text files (*.txt)|*.txt

    Do not use spaces in file masks (like "*.xml"). Also, do not include spaces before and after vertical pipe symbols ("|").

  • Automatic payment allocation. Specifies whether 1C:Drive automatically allocates the imported incoming payments.
  • Import directory. The default folder where the data processor looks for source files.
  • Additional settings. Extra settings that affect the behavior of the data processor. For instance, these can be mappings to the items of the Cash flow items list, which allows 1C:Drive to distinguish between payments from customers, payments to vendors, and other payments.

You can create, edit, and delete data processor settings.
