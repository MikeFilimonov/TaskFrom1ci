Importing bank statements

If your company has a bank account for receiving payments from your customers and for making payments to your suppliers and other parties, you need to register these transactions in 1C:Drive in order to have the clear view of the account's cash in and cash out. Typically, you register bank receipts and bank payments one by one in 1C:Drive.

If you use a bank client system to control transactions of the bank account, you can simplify the task of recording these transactions to 1C:Drive. You can export them from the bank client to a file and then import them to 1C:Drive. As a result, 1C:Drive generates bank receipts and bank payments.

Bank statement import table fields

After 1C:Drive reads data from a bank statement file, it displays the data in the import table.

The idea of the import table is to let you view:

  • The records read from the file
  • The mapping between the record fields and 1C:Drive fields
  • The import status of each record

The mapping table fields are as follows:

  • . The import status. Available values:
    • Red circle. The record is not mapped.
    • Empty green circle. The record is mapped and ready to import.
    • Filled green circle. The record is imported.
  • #. The line number.
  • Received. The date when the bank received the payment document.
  • Document. The document that 1C:Drive created based on the payment from the imported statement. It is read-only.
  • Amount. The amount paid.
  • Company. Your company.
  • Bank account. Your company's bank account.
  • Operation type. The payment operation type. For instance:
    • To supplier
    • To customer
    • Payroll
    • From customer
    • From supplier
    • Other
  • Counterparty. The name of your company's counterparty that made or received the payment.
  • Contract (Optional). The contract between your company and the counterparty.
  • Counterparty bank account. The bank account of the counterparty.
  • Cash flow item. The cash flow item that describes the purpose of the transaction.
  • Expense GL account (Optional). The GL account to write off funds.

Importing bank statement files

To import bank statements to 1C:Drive:

  1. In the bank list, for each bank that provides bank statements, in the Bank statement processing > Settings field, select the bank statement processing settings.

    For more information, see Bank statement processing settings.

  2. Click Cash management > Bank statement processing.
  3. In the Bank to exchange data with field, select the bank.
  4. Click Import statement from file.
  5. Select the source file and click Open.

    1C:Drive reads the file and displays preliminary import results in the import table.

    For more information, see Bank statement import table fields.

    When you select a record in the table, 1C:Drive displays the payment purpose read from the bank statement.

    To hide the records that are marked imported, click Hide imported.

    To show all records, click Show all.

  6. For each record you want to import, do the following:
    1. Check the record's import status.
    2. Skip records that are already imported and concentrate on the records that are mapped and ready to import or not mapped.
    3. If necessary, fill the empty fields in the import table.
    4. If the record is not mapped to 1C:Drive's data, in the Data mapping errors table, set the mapping between the values read from the file and the target attributes and click Apply changes and refresh.

      1C:Drive saves the resulting mapping. To learn how to change it, see Editing bank statement mappings.

  7. Select the records you want to import.

    To select all records, click Mark all.

    To clear selection from all records, click Clear all.

  8. Click Import.

    When creating payment documents based on transactions read from the file, 1C:Drive fills the documents based on the records of the import table and displays the number of documents it created. If there are errors, it continues creating other documents and displays the errors.

  9. If there are mapping errors, for each record where 1C:Drive found a mapping error, click the record, specify the correct mapping, click Apply changes and refresh, and retry importing the data.
  10. Click Cash management > Bank > Bank documents, ensure that all bank documents specified in the bank statement were imported correctly, and post all the documents.
